# Node devDependency Cooldown Bumps + bun Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump `package.json` Node devDependencies to their 7-day-cooldown-eligible versions and remove leftover npm-era artifacts, in one PR driven to CI-green.

**Architecture:** Config-only change. One commit: `bun add -d` applies the eligible bumps (updating `package.json` + `bun.lock` together), `bun remove` drops the vestigial husky/lint-staged, a manual edit deletes the dead `"hooks"` block, and `.bun-version` pins the local toolchain. Local checks verify the dev-tool bumps; PR CI (`test-and-build-on-pr.yml`) is the authoritative check for the electron/@stlite build.

**Tech Stack:** bun 1.3.11, Node 22, electron-builder, @stlite/desktop, commitlint/commitizen, prettier.

---

## File Structure
- `package.json` - devDependency version bumps; removal of husky/lint-staged and the `"hooks"` block.
- `bun.lock` - regenerated by the bun operations.
- `.bun-version` - new file, contents `1.3.11`.

### Task 1: Apply cooldown-eligible devDependency bumps

**Files:**
- Modify: `package.json` (devDependencies)
- Regenerate: `bun.lock`

- [ ] **Step 1: Apply the eligible bumps in one bun add**

Run (caret ranges, updates package.json + bun.lock):
```bash
bun add -d \
  @commitlint/cli@^21.0.2 \
  @commitlint/config-conventional@^21.0.2 \
  @commitlint/cz-commitlint@^21.0.2 \
  @stlite/desktop@^0.101.0 \
  cross-env@^10.1.0 \
  electron@^42.3.2 \
  electron-builder@^26.14.0 \
  prettier@^3.8.3 \
  prettier-plugin-sort-json@^4.2.0 \
  rimraf@^6.1.3
```
Expected: bun resolves and writes the new versions. commitizen is left untouched.

- [ ] **Step 2: Verify package.json reflects the targets**

Run: `grep -E '"(@commitlint/cli|@stlite/desktop|cross-env|electron|electron-builder|prettier|rimraf)"' package.json`
Expected: versions show `^21.0.2`, `^0.101.0`, `^10.1.0`, `^42.3.2`, `^26.14.0`, `^3.8.3`, `^6.1.3`.

### Task 2: Remove bun-migration remnants

**Files:**
- Modify: `package.json` (drop husky/lint-staged + `"hooks"` block)
- Create: `.bun-version`
- Regenerate: `bun.lock`

- [ ] **Step 1: Remove the vestigial husky and lint-staged**

Run: `bun remove husky lint-staged`
Expected: both leave devDependencies and `bun.lock`.

- [ ] **Step 2: Delete the dead `"hooks"` block from package.json**

Remove this block (husky-v4-style key, read by nothing in this repo; pre-commit is owned by `.pre-commit-config.yaml`):
```json
    "hooks": {
        "pre-commit": "lint-staged"
    }
```
Also remove the now-dangling trailing comma on the preceding `config` block if the editor leaves invalid JSON. Verify validity:
Run: `bunx prettier --check package.json || node -e "JSON.parse(require('fs').readFileSync('package.json'))" && echo "valid json"`
Expected: `valid json` (prettier may report formatting; JSON.parse must not throw).

- [ ] **Step 3: Pin local bun version**

Run: `printf '1.3.11\n' > .bun-version`
Expected: `.bun-version` contains `1.3.11`, matching the CI `oven-sh/setup-bun` pin.

### Task 3: Verify the dev-tool bumps locally

**Files:** none (verification only)

- [ ] **Step 1: Lockfile resolves with no drift**

Run: `bun install --frozen-lockfile`
Expected: installs with no "lockfile had changes" error.

- [ ] **Step 2: commitlint v21 loads the config**

Run: `echo "feat: probe" | bunx commitlint && (echo "nope" | bunx commitlint; test $? -ne 0 && echo "rejects-bad-ok")`
Expected: the valid message passes (exit 0) and the malformed one is rejected (`rejects-bad-ok` printed). Confirms `commitlint.config.cjs` + config-conventional load under v21.

- [ ] **Step 3: prettier 3.8 + sort-json plugin load**

Run: `bunx prettier --version && bunx prettier --check .bun-version >/dev/null 2>&1; echo "prettier-ran:$?"`
Expected: prints a 3.8.x version and `prettier-ran:` with any exit code (the point is the binary + plugin load without a module error).

- [ ] **Step 4: cross-env v10 and rimraf v6.1 execute under Node 22**

Run:
```bash
bunx cross-env PROBE=ok node -e "process.exit(process.env.PROBE==='ok'?0:1)" && echo "cross-env-ok"
mkdir -p /tmp/rimraf-probe/x && bunx rimraf /tmp/rimraf-probe && test ! -d /tmp/rimraf-probe && echo "rimraf-ok"
```
Expected: `cross-env-ok` and `rimraf-ok`.

- [ ] **Step 5: (Optional, network-permitting) stlite build smoke**

Run: `timeout 600 bun run dump 2>&1 | tail -20 || echo "dump-skipped (network/time)"`
Expected: either the dump artifacts are produced, or it is skipped. NOT a blocker — CI is authoritative for the build. Do NOT run `bun run dist` locally (OS-specific packaging).
