# Standards

このディレクトリは標準化された開発規約の置き場です。
コーディング規約・テスト規約・セキュリティ規約・CI-CD規約・ディレクトリ規約など、プロジェクト全体で遵守すべき開発標準を格納します。

## ドキュメント一覧

- [directory-structure.md](./directory-structure.md): ディレクトリとファイルの配置規則、およびドキュメント/ルールの配置原則を定める。
- [python-coding.md](./python-coding.md): Pythonの型ヒント、命名、関数設計、バリデーション、エラー処理、コードスタイルの規約を定める。
- [python-testing.md](./python-testing.md): pytestを前提としたテストコードの設計、配置、パラメーター化、モックの規約を定める。
- [python-security.md](./python-security.md): 製品コードと依存ライブラリのセキュリティリスクチェックおよび脆弱性対策の規約を定める。
- [ci-cd.md](./ci-cd.md): GitHub Actionsの基本ルール、ワークフロー、composite actionの規約を定める。
- [local-development.md](./local-development.md): git、gitignore、pre-commit、commitlintによるローカル開発環境の規約を定める。
- [package-management.md](./package-management.md): poetryおよびnpmによる依存パッケージ管理の規約を定める。
