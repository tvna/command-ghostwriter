# Redesign B — 2ペイン・エディタ（デザイン格納物）

Streamlit 版 Command Ghostwriter を Vercel へ移行するにあたって作成した、
**新 UX（Redesign B / 2ペイン・エディタ）のデザインと実装見本**を格納したものです。
Claude Design（claude.ai/design）でのデザイン作業の成果物を、本リポジトリに取り込みました。

> **重要 — このフォルダは参照用です。** 既存の Streamlit アプリ（リポジトリ直下の
> `app.py` ほか）とはまだ**結合していません**。本体への統合は別フェーズで対応します。
> ここにあるのは「何を・どう作るか」を確定させた設計の土台です。

## 中身

| パス | 内容 |
| --- | --- |
| `app/` | **実装見本** — Vite + React + TypeScript で組んだ新 UX 一式。`cd app && npm install && npm run dev` で起動。詳細は `app/README.md`。 |
| `project/` | **デザインシステム + プロトタイプ** — トークン（色/タイポ/余白）、再利用コンポーネント、`project/redesign/` の元 HTML プロトタイプ、アセット（ブランドマーク・インフラ系アイコン）。 |
| `chats/` | **設計経緯の全ログ** — UX ストーリーマップ → 3 案比較 → B 案決定 → エディタ/空状態/バリデーション/ライブ編集/テンプレートライブラリ、と詰めていった背景情報。意図はここに残っています。 |
| `HANDOFF.md` | Claude Design が出力した元のハンドオフ説明。 |

## 設計の要点（結合フェーズの前提）

- **方向性**: 入力（データ定義）と出力（手順書）を常時併置する**2ペイン・エディタ**。デスクトップ専用（モバイル対象外）。
- **暗黙フォーマット判定バグの解消**: TOML/YAML/CSV を**明示選択**＋誤判定時は実パースエラーで該当行をハイライトし、実際にパースできる形式を提案。
- **出力思想の刷新**: スニペット投入型から、**CLI をコードブロックに収め前後に人の手順を挟む Markdown 手順書**型へ。右ペイン既定は「手順書」、ほかに Raw / Visual Debug。
- **ライブ編集**: 左ペインの編集が右ペインへ即時反映（`app/src/lib/engine.ts` の軽量 TOML/YAML/CSV パーサ + Jinja 風レンダラ）。
- **画面遷移**: 空状態 → テンプレートライブラリ → エディタ。ハッシュルーティングで戻る/進む・リロード復元に対応。

## 実装スタックについて

元のプロトタイプはブラウザ内 Babel/UMD の試作でしたが、本格実装として
**Vite + React + TypeScript** に作り直しています（型チェック・本番ビルド確認済み）。
ビジュアルはすべてデザインシステムの CSS カスタムプロパティ（`--cg-*`）で表現しており、
Streamlit 由来のブランド（ダークテーマ / `#FF4B4B` / Source Sans 3・Source Code Pro）を踏襲します。
