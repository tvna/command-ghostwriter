# UI 刷新: 2 ペイン・ライブエディタへの置換（Vercel 移行・第一弾）

Closes #<ISSUE_NUMBER>

## 概要

Streamlit の 5 タブ UI を、**2 ペイン・ライブエディタ**へ置換します。左で入力（データ + テンプレート、フォーマット明示選択）、右で出力（手順書 Markdown をデフォルト、Raw / Visual Debug 併設）を常時併置し、編集を即時反映するライブ生成を実現します。本 PR は移行の第一弾として、UI シェル・ライブ生成エンジン・初期テンプレート 6 種を導入します。

> 思想の転換: 「CLI スニペットを投入するツール」から、**「CLI をコードブロックに収め、前後に人の手順を挟む Markdown 手順書を生成するツール」** へ。

## 変更点

### UI / 画面
- **空状態（オンボーディング）**: 2 ファイルの関係を図解、サンプル / アップロード / ライブラリの 3 入口。
- **2 ペイン・エディタ**: 左=入力（データ定義 / テンプレート、TOML/YAML/CSV 明示選択）、右=出力（手順書 / Raw / Visual Debug）。VS Code スタイルの固定フレーム + ペイン内部スクロール。
- **テンプレートライブラリ**: 左カテゴリ + 右カードグリッド。Cisco スイッチポート / YAMAHA ルータ / Linux 初期セットアップ / DNS ゾーン / 障害対応手順書（キャンパス / プロキシ）。
- **詳細設定**: 独立タブを廃しモーダルへ。
- **URL ルーティング**: `#/`・`#/library`・`#/new`・`#/t/<id>`。戻る/進む・リロード復元に対応。

### バグ修正（最優先）
- **暗黙フォーマット判定の撲滅**: フォーマットを明示選択。誤選択・パース失敗時は **該当行ハイライト + 具体的メッセージ + 自動判定（auto-detect）による修復導線** を表示し、動作不能を防止。

### ライブ生成エンジン
- 軽量 **TOML / YAML / CSV パーサ**（array-of-tables `[[x]]`、インデント YAML、CSV ヘッダ/行に対応）。
- 簡易 **Jinja レンダラ**（`{{ var }}` / `| join` / `{% for %}` / `{% if/else %}` / `loop_index`）。
- Jinja の **trim_blocks / lstrip_blocks をデフォルト ON** とし、ループ起因の余計な改行を圧縮。
- 編集 → パース → 出力・変数検出・バリデーションを毎編集で再計算。

### その他
- ライブ生成化に伴い「生成」ボタンを撤去。
- 既存の絵文字アイコンを廃し、ターミナル顔のゴーストマーク + インフラ系 SVG アイコンセットへ刷新。
