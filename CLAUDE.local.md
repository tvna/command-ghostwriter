# CLAUDE.local.md

Project-level operating rules that supplement the APM-generated `CLAUDE.md`.

This file is intentionally committed (it is deliberately **not** added to
`.gitignore`) so the rules below also apply in the remote "Claude Code on the
web" environment, which clones the repo fresh per session. The rules live here
rather than in `CLAUDE.md` because that file is generated by `apm compile` and
is overwritten on every regeneration.

## Retrospective follow-ups

- When a merged PR is itself a retrospective follow-up — it closes an action
  issue spun off from a parent retrospective it references — record its
  repair-free-merge review as a note on that parent retrospective issue instead
  of opening a new retrospective issue, so the audit trail stays in one place
  and the retro chain stays bounded. Only a genuinely new (non-follow-up) merge
  opens its own retrospective issue.

## Release-archive preflight

- Before authoring any extraction logic for a release artifact (tar.gz, zip,
  etc.), run `scripts/inspect_release_archive.sh <pinned-url> --sha256 <hex>
  [--expect-path <member>]` and encode the observed layout (nesting,
  `--strip-components`, member paths) in the code and its comments. Never
  write extraction paths from assumption; the apm nested-directory failure on
  PR #391 (exit 127) is the bug class this prevents. Refs #392, #393.
